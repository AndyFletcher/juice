Juice the javascript web framework

Stop writing your logic twice. Juice removes the context switch between client and server code by using the same language — and code — in both places.

Why use Juice?

  • Javascript is fun.

    If you enjoy working in javascript, then you'll enjoy Juice. If you don't, you probably won't get much out of it.

  • Lower barriers.

    With your client-side and server-side code in the same language, your development won't hit as many communication blocks or gear changes.

  • Planned Share code between client and server.

    Common helpers and validation routines written only need writing once and can be used in your controllers, templates and client-side code.

Why not Juice?

  • It's not production-ready.

    We started writing Juice to scratch an itch. If you want to play around or use it for a vanity site or a toy project then great, but if you're looking to launch your VC-funded app next month you might want to pick something else.

  • You need Enterprise labels.

    If you have to sell this your boss then Juice probably isn't for you.

  • You hate javascript.

    We understand, though we think there's a good chance your hatred of it comes from dealing with browser inconsistencies, so we think you should give it a try anyway.

Installation

Mac OS X (10.5+)

  1. Download and install Juice 0.1 (2.1MB dmg)
  2. Open Terminal and create a project: juice init helloworld
  3. Start the server: ./helloworld/script/server
  4. Visit your app: open http://localhost:3000/

Windows (XP or newer)

  1. Download and install Juice 0.1 (3.4MB msi)
  2. Open a command prompt and create a project: juice init helloworld
  3. Start the server: helloworld\script\server.bat
  4. Visit your app at http://localhost:3000/
