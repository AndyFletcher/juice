Juice the javascript web framework

Stop writing your logic twice. Juice removes the context switch between client and server code by using the same language — and code — in both places.

Why use Juice?

  • Javascript is fun.

    If you enjoy working in javascript, then you'll enjoy Juice. If you don't, you probably won't get much out of it.

  • Lower barriers.

    With your client-side and server-side code in the same language, your development won't hit as many communication blocks or gear changes.

  • Planned Share code between client and server.

    Common helpers and validation routines written only need writing once and can be used in your controllers, templates and client-side code.

  • Planned Document-oriented database out of the box.

    Or a relational database, if that's your bag. We prefer the first choice, even though we've never tried it.

Getting started

The easy way is to wait a couple of weeks until we've sorted out packaging up binaries. The hard way is to install several things, some from source. If you're that desperate, come ask us for help in IRC.
